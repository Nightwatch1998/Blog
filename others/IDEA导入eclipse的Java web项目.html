      # IDEA导入Eclipse的Java web项目

      最近工作上要维护一个老的Java Web项目（没注释，谁来救救我！），拿到的源代码不完整，分别是Java Web项目的src目录和WebContent（WebApp）目录，刚开始把两部分代码组合为完整的项目就浪费了很多时间。

      查资料后了解了Java Web项目的基本目录结构，起初尝试在Eclipse中配置运行，但总是有问题，血压直接上来了，后面再尝试吧。对IDEA更熟悉一些，界面UI更为人性化，操作过程中遇到了很多坑，这里总结一下。

      # 目录结构

      Eclipse的项目结构一般为

      • src
      • WebContent
      • .classPath
      • .project

      # 导入

      首先将项目整体导入，然后在根目录新建一个out文件夹，作为项目的输出文件夹

      然后配置项目的项目结构

      # Project

      选择jdk版本，以及输出文件目录

      # Modules

      将src目录标注为源文件夹

      # Libraries

      添加本地依赖所在的目录

      # Facets

      添加web的facets，并核对web.xml的配置文件路径是否与项目中的WebContent目录一致，配置Web资源目录

      # Artifacts

      添加lib文件夹

      # 运行
